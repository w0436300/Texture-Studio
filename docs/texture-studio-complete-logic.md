# Texture Studio · 3D 材质文字生成器 — 完整逻辑说明

> 本文档为产品/工程合一的**最终逻辑约定**：覆盖「**逐字 3D + 独立拖拽/旋转 + 逐字材质**」「**Three 轨免费 vs AI 贴图轨按字缓存**」、整图 Gemini、交互与 **glyph 软限制**。实现以 `lib/textures.ts` 的 `isAiTextureMaterial` 等为真源；不必与旧版草案逐字相同，但**语义以本文为准**。

---

## 一、目标与总原则

### 1.1 主流程（以 Three 逐字场景为准）

1. **输入**：用户输入一段文字。  
2. **拆字**：每个需要可视化的 **glyph**（不含空格等占位规则见 §2）对应 **独立 3D 挤出几何体**（如 `TextGeometry` + `Group`）。  
3. **交互**：每个字母 **独立** 支持 **拖拽摆放位置**、**旋转**（及后续可扩展缩放）；**每个字母可绑定不同材质 id**（与「材质模式」或混合分配一致）。  
4. **材质两轨**（成本边界清晰）：  
   - **Three 轨（免费）**：金属、玻璃、全息、霓虹、陶瓷、果冻、乳胶等 —— 用 **Three.js PBR / 程序化参数** 直接在 mesh 上渲染，**不调**图像生成 API。  
   - **AI 贴图轨（按字 + 材质计费一次）**：苔藓、绒毛、毛毡、木纹、大理石纹、蜡质等 —— 归为 **「难以仅靠 Three 稳定还原的有机 / 强微观纹理」**，通过 **Gemini 等图像 API** 生成 **方形 albedo 贴图**，贴到该字的挤出 mesh 上；结果按 **`mat_{Unicode码点}_{材质id}`** 写入 **localStorage，永久缓存**；**同一字母 + 同一材质 id 第二次起只读缓存、不调接口**（用户侧「只花一次钱」语义）。  
5. **整句 Gemini 海报**（2D 位图）可与本流程 **并存**，但缓存 key、统计规则 **不得与逐字 3D 混用**（见 §十三）。

### 1.2 与「档位」的关系

| 档位     | 含义 |
|----------|------|
| **Three 实时** | 本字当前材质 ∈ Three 轨 → 仅 GPU，$0。 |
| **AI 贴图** | 本字当前材质 ∈ AI 轨 → 缺缓存时调模型生成贴图；有缓存则 $0。 |

工程上 **AI 贴图** 由用户显式触发「补全缺失贴图」按钮批量拉取（避免每改一个笔画就自动扣费）；策略可调为「首次进入含 AI 材质的组合时自动补一行」等，但 **计费语义** 仍以 **字 + 材质** 缓存命中为准。

## 二、字符数：从「硬 8」改为「软限制」

### 2.1 为什么要软限制

性能与交互成本大致随 **可见字形数量** 线性（甚至超线性）增长：Mesh 数、射线检测、InstancedMesh 总实例数、显存。  
**硬砍 8 个字**对「一句短句、两词标题」偏紧；**完全不限**又容易在低端设备上踩雷。  
因此采用 **软限制 + 自动降级 + 可选硬上限**，而不是「超过 8 禁止输入」。

### 2.2 字符怎么数（全局统一）

| 概念 | 规则 |
|------|------|
| **渲染单位（glyph）** | 用户输入中**需要生成 3D Mesh 的字符**。**空格**不生成 Mesh，但可参与**排版占位**（宽度见布局章节）。 |
| **标点** | 每个标点算 **1 个 glyph**（与字母同等参与计数与性能预算）。 |
| **中英文** | 每个汉字/假名/韩文音节等 **1 个 glyph**（与「8 个字母」信息量不可比，这是产品文案要说明的点）。 |
| **不可见字符** | 建议过滤或合并（如连续空格压成占位规则），避免统计与体验不一致。 |

记：

- `G = 可见 glyph 数量`（**不含空格**）。

### 2.3 软限制分档（推荐默认）

以下分档与 **自动降级策略** 绑定（见 §7.4、§10）：

| 区间（G） | 名称 | UI 行为 | 渲染/性能行为 |
|-----------|------|---------|----------------|
| **1–8** | **推荐区** | 无提示 | 全量效果：默认 Instancing 预算、完整粒子与灯光动画（若开启） |
| **9–12** | **可接受区** | **非阻塞提示**（如：「字符较多，已自动降低毛发/苔藓细节以保证流畅」） | 自动降低每 glyph 的 Instancing 上限、略减 bevel/曲线细分 |
| **13–16** | **吃力区** | 提示 + 可选「仍继续」 | 进一步降低实例数、默认关闭或简化背景粒子与动态灯光 |
| **17–24** | **极限区** | 明确提示「可能影响帧率与内存」；建议截断或拆分 | 强制应用「低面数 + 低实例」配置；可考虑仅对**选中字**使用高细节 |
| **> 24** | **硬上限（建议）** | **禁止继续**或要求用户**显式勾选**「我了解可能卡顿/崩溃」的**高级模式** | 仅用于桌面调试或明确标注实验性 |

> **说明**：表格中的数字为**推荐工程默认**；产品可将「硬上限 24」改为 20/32，但**必须在文档与 UI 固定一处**，避免前后端各写各的。

**原则一句话**：`8` 不再是「法律」，而是**推荐舒适区**；超出后**仍能玩**，用**自动降级**换流畅，而不是简单禁止。

## 三、文字处理管线

### 3.1 拆字

- 将规范化后的字符串拆成数组，**每个 glyph 一个节点**：独立 `group` / `mesh`、独立变换与材质句柄。

### 3.2 空格

- **不创建几何体**；在布局步中占用 **可配置水平间距**（如 `0.2～0.4` 世界单位，随全局字号缩放），避免两字粘在一起。

### 3.3 修改文字

- 销毁旧 glyph 的 Mesh/材质/实例化附属物，按新串重建。  
- **状态复用策略（建议）**：按**新串从左到右的索引**尝试继承旧材质/变换；多出来的新字用默认；减少的字删节点。

## 四、3D 几何：字体 → 挤出

### 4.1 资源

- 使用 **Three.js `FontLoader` + `TextGeometry`（或等价 API）**；字体为 JSON 曲线（如 typeface 系），**CDN 或自带静态资源**，并注意**商用授权**与**子集**（只打包常用字符以控体积）。

### 4.2 关键参数（与性能联动）

- `size`：建议约 `1.2–1.5`（与场景尺度统一）  
- `height`（挤出厚度）：`0.4–0.5` 量级，厚一点材质更易读  
- `bevelEnabled: true`；`bevelThickness / bevelSize` 在 **推荐区** 可偏高；**高 G 区** 应随降级减小  
- `curveSegments`：高 G 时**必须**下调，控制三角面数  

生成后 `computeBoundingBox()`，将几何**平移到以自身中心为原点**，便于旋转/拖曳与散点布局。

### 4.3 面数与软限制的联动

- 三角面数 ↑ → Raycaster 与**表面发束采样**都更贵。  
- **分档**见 §2.3：高 G 时优先砍 **curve / bevel**，而不是直接砍「材质种类」。

## 五、初始布局：散落

（逻辑保持原方案精神，与实现细节可调。）

- 在 `[-W, W]` 内依序铺开基准 x，加小随机；y 可奇偶错层；z 小范围前后；`rotation.z` 小角度随机；`scale` 可 `0.9–1.1` 轻随机。  
- **防重叠**：可用包围盒+最小间距迭代，或简单规则+回调「若重叠则重 roll」（次数上限）。

## 六、材质策略总览

### 6.1 两轨制（工程单一来源）

- **实现**：`lib/textures.ts` 中 `isAiTextureMaterial(id)`（及 `AI_TEXTURE_MATERIAL_IDS` 集合）为 **唯一真源**；UI「是否展示 AI 补图按钮」、服务端 `/api/generateHdMaterial` 校验、Three 是否尝试加载 `localStorage` 贴图，均须与此一致。  
- **Three 轨**：`MeshPhysicalMaterial`（等）+ 预设参数；可扩展 `CanvasTexture` / 噪声 `bumpMap`，仍不调云端图像模型。  
- **AI 轨**：仅对 **有机 / 强微观** 材质生成 **无缝方形 tile**，作为 `map` 叠在挤出几何上；**不按帧**调用，按 **(码点, 材质 id)** 持久缓存。

### 6.2 当前产品划定的 AI 贴图材质 id

以下材质 id 走 **AI 贴图轨**（其余 concrete id 均为 Three 轨）：

| 材质 id | 说明 |
|---------|------|
| `moss` | 苔藓 |
| `plush` | 绒毛 |
| `felt` | 毛毡 |
| `wood` | 木纹 |
| `marble` | 大理石纹 |
| `wax` | 蜡质 |

`random` / `mixed` 为入口态：判断时用 **`resolveTextures` 之后** 落到每个 glyph 上的 **具体 id**。

### 6.3 每 glyph 独立材质实例

- **禁止**多字共用同一 `Material` 引用（除非只读且从不改 uniforms）。每字 `clone()` / `new` 材质实例，便于独立贴图与参数。

## 七、苔藓 / 毛绒等的实现路径（Instancing vs AI 贴图）

### 7.1 当前工程默认

- **苔藓、绒毛、毛毡、木、大理石、蜡**等：优先 **AI 方形贴图 + localStorage**（见 §一、§八），在挤出 mesh 上作 `map`，开发与迭代成本可控。  

### 7.2 可选增强（文档级）

- 若未来要 **纯几何发束 / InstancedMesh** 苔藓：在 glyph 表面**面积加权**采样，沿法线发细柱，打成少量 `InstancedMesh`；与 **软限制分档**（§2.3、§十）联动降低实例预算。  
- 与 AI 贴图 **二选一或分层**（底 PBR + 顶 AI 细节）需产品单独拍板。

### 7.3 实现注意（若做 Instancing）

- 三角上采样需**面积加权**；细笔画处三角形少时可能发束稀疏 —— 需**细分**或**表面重采样**策略。

## 八、AI 贴图生成与缓存（逐字、按材质）

### 8.1 触发

- 当前 UI：**仅当**文案下至少一字解析后的材质 ∈ `isAiTextureMaterial` 时，展示「补全 AI 贴图」类按钮；**仅对缺失缓存的 (码点, 材质 id)** 调用 `/api/generateHdMaterial`。  
- 后端：Gemini（或 env 配置的图像模型）；提示词按材质 id 分支（`lib/hd-material-prompt.ts`）。  

### 8.2 产物语义（已定）

- **主路径**：**方形 1:1 albedo 风格贴图**，作为 Three `MeshPhysicalMaterial.map` 使用；与挤出 UV 为 **工程折中**，不承诺物理无缝，但要求 **tile 可读、无字剪影**。

### 8.3 缓存 key

- **`mat_{Unicode码点}_{材质id}`**（例：`mat_72_moss` = `H` + 苔藓）。  
- **无 TTL**：不主动过期；同一 key **永命中同一张**直至用户清站点数据。  
- 若未来要「同款多随机」，必须把 **variant / seed** 编入 key。

### 8.4 存储介质

- 现工程用 **localStorage** 存 base64 串；体积大时产品可迁移 **IndexedDB**，key 规则保持不变。

## 九、交互

- **拾取**：`Raycaster`；拖拽时**临时禁用**轨道旋转，避免和拖字冲突。  
- **拖拽**：在固定深度平面与射线求交，避免 z 乱飘。  
- **旋转/缩放**：如滚轮绕 z、Shift+滚轮 scale，并 clamp。  
- **浮层面板**：材质、高清、删除；点空白失焦。  
- **触摸**：首版可声明仅键鼠，或给平移/双指等映射。

## 十、性能与画质的「自动降级」表（与 §2.3 对应）

| 项目 | G≤8 | 9–12 | 13–16 | 17–24 |
|------|------|------|--------|--------|
| `curveSegments` | 高 | 中 | 中低 | 低 |
| bevel 强度 | 高 | 中 | 低 | 最低或关 |
| 单 glyph 毛发/苔实例预算 | 100% | ~70% | ~45% | ~25% |
| 背景粒子/动态灯光 | 开 | 可开 | 减/关 | 关 |
| 仅选中高细节 | 否 | 可选 | 建议 | 建议 |

> 表内百分比为**相对默认预算**，具体数值在实现中配置成常量，便于 A/B 与监控。

## FAQ、混合材质与「单字组合」

### FAQ.1 混合材质的两层含义

| 区域 | 混合材质表现 |
|------|----------------|
| **整图 Gemini（已弃用为主流程）** | 历史能力：`/api/generate` 曾可返回单张 PNG。**当前默认**：ASCII 走 **Three 逐字组合**；整图 Gemini 不再由 `generate` 调用。 |
| **Three.js（代码库内）** | `resolveTextures("mixed", text)` 等仍可用于 **自建入口**（每字 mesh + PBR + 可选单字 AI tile）；**主站默认 UI 已收束**，不再把 Three 预览作为与整图并列的主操作区。 |
| **本地 SVG (mock)** | **逐字** 矢量片段 + 可拖动；用于「可编排单字」的示意路径，与整图海报 **视觉不同源**。 |

### FAQ.2「单个字母组合」在现有产品里的做法

1. **照片级 + 混合 + 与参考图类似的整句海报**: 选 **混合材质** + `Generate`（Gemini）→ **上方单张整图**；混合模式 prompt 已强化「每字不同实体材质 + 可选纯白底」描述。  
2. **必须在浏览器里拖动单个字母**: 单张 Gemini PNG **不可拆层** → 请用 **本地 SVG / mock** 的交互预览（风格化示意）。  
3. **Three + 单字 AI tile（主路径）**: ASCII 文案下首页 **Three 组合** 为默认主预览；有机档字可走 **补全 AI 贴图**（`mat_{码点}_{id}` 永久缓存）。整图 Gemini 海报 **已不作为** `/api/generate` 的默认输出。  
4. **若坚持「整图也要拆成单字编辑」**: 单张位图 **做不到**;替代是 **每字单独调一次** 图像生成再在画布上拼（成本高,可另做产品入口）。

## 十三、与当前 Web 应用（Next.js + Gemini 整图）的关系

- **整句一张图**：顶部 Gemini 海报为 **单张位图**，**不能**拆成逐字图层；其缓存 / prompt 与 **逐字 3D** 无关。  
- **逐字 3D（可选 / 代码保留）**：Three 预览为 **每字独立 mesh + 拖拽旋转**；材质按 **§六** 两轨执行；**AI 贴图** 仅服务 **AI 轨材质** 的 **单字 tile**，key 见 §8.3。**主站默认 UI 以整图混合为主**,不再并列展示 Three 条。  
- **禁止**混用 key（整图 ≠ `mat_*`）；glyph 软限制若同时启用多条路径,应 **分别说明** 统计口径。
